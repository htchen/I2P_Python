## 第 1 題：可變 vs. 不可變資料型態

請回答下列問題：

(a) 請舉出 Python 中**兩種「不可變（immutable）」**的資料型態或資料結構。

(b) 請舉出 Python 中**兩種「可變（mutable）」**的資料型態或資料結構。

(c) 請說明「可變」與「不可變」資料在程式設計上有什麼差異？並舉一個因為使用可變資料而可能產生 bug 的情境。

### 參考答案

(a) 不可變（任選兩個即可）：
- `int`、`float`、`bool`、`str`、`tuple`、`frozenset`

(b) 可變（任選兩個即可）：
- `list`、`dict`、`set`、自訂的 class instance

(c) **差異：**
- 不可變資料一旦建立後，內容無法被修改；任何「修改」操作都會產生一個新的物件。
- 可變資料可以在原地（in-place）被修改，所有指向該物件的變數都會看到變化。

**Bug 情境範例：** 用可變物件當作函式的「預設參數」：

```python
def add_item(item, bag=[]):   # ❌ 預設值 [] 只會被建立一次
    bag.append(item)
    return bag

print(add_item("apple"))   # ['apple']
print(add_item("banana"))  # ['apple', 'banana']  ← 出乎意料！
```

或是「兩個變數指向同一個 list」造成的副作用：

```python
a = [1, 2, 3]
b = a            # b 和 a 指向同一個 list
b.append(4)
print(a)         # [1, 2, 3, 4]  ← a 也被改了
```

如果改用 tuple，則上述問題不會發生，因為 tuple 不允許被修改。

## 第 2 題：純函式（Pure Function）

什麼是「純函式（Pure Function）」？為什麼 `sorted(my_list)` 比 `my_list.sort()` 在程式設計上更接近「純」的概念？這種寫法在維護程式碼時有什麼好處？

### 參考答案

**純函式的定義：** 一個函式如果同時滿足以下兩個條件，就稱為純函式：

1. **相同的輸入永遠得到相同的輸出**（不依賴外部狀態，例如全域變數、時間、隨機數）。
2. **沒有副作用（side effect）**：不會修改傳入的參數、不會修改全域變數、不會做 I/O（印出、寫檔、發網路請求）等。

**為什麼 `sorted()` 比 `.sort()` 更「純」？**

| | `sorted(my_list)` | `my_list.sort()` |
|---|---|---|
| 是否修改原始 list | 否（產生新 list） | 是（in-place 修改） |
| 回傳值 | 排序後的新 list | `None` |
| 對外部狀態的影響 | 無 | 有（原 list 被改變） |

`sorted()` 不會改變傳入的 `my_list`，符合「無副作用」這個性質，因此比較接近純函式。

**維護程式碼的好處：**

1. **容易推理（reasoning）：** 看到純函式的呼叫，不用擔心它偷偷改了別的變數。
2. **容易測試：** 同樣輸入永遠同樣輸出，單元測試只需要比對回傳值。
3. **容易除錯：** Bug 不會擴散到呼叫點以外的地方。
4. **容易並行（parallel）：** 沒有共享狀態，多執行緒不會互相干擾。

## 第 3 題：錯誤處理的設計觀念

程式中遇到「網路斷線」、「API 回傳 404」、「JSON 格式錯誤」等情況時，為什麼通常使用 `try / except` 而不是用 `if` 來事先判斷？請比較這兩種風格的差異與各自的適用時機。

### 參考答案

**為什麼用 `try / except`？**

像「網路是否暢通」「API 回傳是否正確」「檔案內容是否合法 JSON」這類狀況，**無法在執行前事先檢查**——
- 即使你先 `if` 檢查網路通了，下一秒也可能斷線。
- 你無法在不解析 JSON 的情況下，事先知道它是否合法。

這類「結果只有真的去做才會知道」的情境，使用 `try / except` 才能正確捕捉錯誤。

**兩種風格的名稱（常見說法）：**

- **LBYL（Look Before You Leap）：** 用 `if` 事先檢查條件再行動。
- **EAFP（Easier to Ask for Forgiveness than Permission）：** 直接做，失敗了再處理。Python 偏好這種風格。

**比較：**

| | `if` 事先判斷（LBYL） | `try / except`（EAFP） |
|---|---|---|
| 適用對象 | 條件可預先確定（例如 `if x > 0`、`if key in dict`） | 條件**不可預測**（網路、檔案、外部 API） |
| 競態條件（race condition） | 容易發生（檢查完到使用之間狀態可能改變） | 不會（直接嘗試動作） |
| 程式碼可讀性 | 「正常邏輯」與「錯誤處理」混雜 | 兩者分開，主流程乾淨 |
| 效能 | 多一次檢查 | 沒錯誤時較快，有錯誤時稍慢 |

**簡單原則：**
- 純粹的數學/邏輯條件 → 用 `if`。
- 與外部世界互動（網路、檔案、使用者輸入、解析資料） → 用 `try / except`。

## 第 4 題：函數式三劍客（map / filter / reduce）

請用一句話分別說明 `map`、`filter`、`reduce` 三者的用途與差異。並說明為什麼這種寫法被稱為「宣告式（declarative）」，它與傳統 `for` 迴圈在「描述程式」的方式上有什麼不同？

### 參考答案

**三者的用途：**

- **`map(f, xs)`：** 對序列裡**每一個元素**套用函式 `f`，產生**等長**的新序列。（一對一轉換）
- **`filter(f, xs)`：** 用條件函式 `f` 過濾序列，**保留** `f(x)` 為真的元素，產生較**短**的序列。（篩選）
- **`reduce(f, xs)`：** 用二元函式 `f` 將序列「累積」成**單一個值**（例如總和、最大值、累乘）。（聚合）

| | 輸入 | 輸出長度 | 概念 |
|---|---|---|---|
| `map` | 序列 + 轉換函式 | 與原本相同 | 變形 |
| `filter` | 序列 + 條件函式 | ≤ 原本 | 篩選 |
| `reduce` | 序列 + 結合函式 | 1（單一值） | 聚合 |

**為什麼是「宣告式」？**

- **指令式（Imperative，for 迴圈）：** 一步一步告訴電腦**怎麼做**——「開一個空 list、走訪每個元素、判斷條件、append 進去……」。
- **宣告式（Declarative，map / filter / reduce）：** 只告訴電腦**要什麼結果**——「我要把這串資料每個都乘 2」「我要保留評分大於 4 的」。怎麼跑迴圈、用什麼順序，由語言/函式庫負責。

**例子比較：**

```python
# 指令式：描述「怎麼做」
result = []
for x in nums:
    if x > 0:
        result.append(x * 2)

# 宣告式：描述「要什麼」
result = list(map(lambda x: x * 2, filter(lambda x: x > 0, nums)))
```

**好處：** 程式碼更短、意圖更明顯、不容易寫錯迴圈邊界，也更容易被優化（例如平行化）。

## 第 5 題：進位制轉換

請完成下列轉換（需寫出計算過程）：

(a) 將十進位 `156` 轉換為**二進位**與**十六進位**。

(b) 將二進位 `1011 0110` 轉換為**十進位**與**十六進位**。

(c) 將十六進位 `0x2F` 轉換為**十進位**與**二進位**。

(d) 為什麼程式設計中常用「十六進位」來表示二進位數值？（請從「易讀性」的角度說明。）

### 參考答案

**(a) 十進位 156 → 二進位、十六進位**

除 2 取餘（由下往上讀）：
```
156 ÷ 2 = 78 ... 0
 78 ÷ 2 = 39 ... 0
 39 ÷ 2 = 19 ... 1
 19 ÷ 2 =  9 ... 1
  9 ÷ 2 =  4 ... 1
  4 ÷ 2 =  2 ... 0
  2 ÷ 2 =  1 ... 0
  1 ÷ 2 =  0 ... 1
```
**二進位：`1001 1100`**（驗算：128 + 16 + 8 + 4 = 156 ✓）

由二進位每 4 bit 轉成 1 個十六進位數字：
- `1001` = 9
- `1100` = C（12）

**十六進位：`0x9C`**（驗算：9 × 16 + 12 = 156 ✓）

---

**(b) 二進位 `1011 0110` → 十進位、十六進位**

依各 bit 的權重相加：
```
1 0 1 1 0 1 1 0
128 64 32 16  8  4  2  1
```
= 128 + 32 + 16 + 4 + 2 = **十進位 `182`**

每 4 bit 拆開：
- `1011` = 11 = B
- `0110` = 6

**十六進位：`0xB6`**（驗算：11 × 16 + 6 = 182 ✓）

---

**(c) 十六進位 `0x2F` → 十進位、二進位**

依位數權重：
- 2 × 16¹ + 15 × 16⁰ = 32 + 15 = **十進位 `47`**

每個十六進位數字展開成 4 bit：
- `2` = `0010`
- `F` = `1111`

**二進位：`0010 1111`**（驗算：32 + 8 + 4 + 2 + 1 = 47 ✓）

---

**(d) 為什麼用十六進位表示二進位？**

- 二進位數字太冗長，例如 32-bit 數值可能要寫 32 個 0/1，難讀也容易抄錯。
- **1 個十六進位數字 = 恰好 4 個二進位 bit**（因為 2⁴ = 16），所以二進位與十六進位之間可以**逐段直接互轉，不必經過十進位計算**。
- 因此十六進位**長度只有二進位的 1/4**，但又能保留每個 bit 的資訊。
- 例如表示一個 byte（8 bit）的記憶體位址或顏色值：
  - 二進位：`11111111` ← 不易閱讀
  - 十進位：`255` ← 失去 bit 對應
  - 十六進位：`0xFF` ← 又短、又能對應 bit pattern
- 這也是為什麼網頁顏色（`#FF8800`）、記憶體位址、bitmask 都習慣用十六進位來表示。
